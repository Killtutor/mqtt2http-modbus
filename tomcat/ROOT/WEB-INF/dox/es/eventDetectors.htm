Visión general

   El propósito de un detector de evento es determinar si el valor de un punto satisface una o más condiciones relacionadas, y generar un evento que puede ser adecuadamente manejado. Un punto puede tener un número ilimitado de detectores de eventos. Las formas en que los eventos pueden ser detectados dependen del tipo de datos del punto, y por ello sólo se enumeran los tipos de detectores de eventos soportados por el punto.

   Los detectores de eventos pueden detectar un gran número de condiciones, pero sólo se aplican a un único punto. Si se requieren condiciones a través de múltiples puntos o condiciones que incluyan eventos programados, se deben utilizar detectores de eventos compuestos. (Ver la documentación "Detector de eventos compuestos" para más información).

Un punto puede tener tantos detectores como sea necesario, y puede tener múltiples detectores del mismo tipo. (Es decir, múltiples detectores de límite superiores o inferiores.)

   Para agregar un nuevo detector debe seleccionar el Tipo y hacer clic en el icono . Nota: para guardar los detectores de eventos debe hacer click en el botón "Guardar" ubicado en la parte inferior de la página. Para borrar un detector de eventos existente hacer click en el icono asociado al detector.

Cada detector tiene asociado un Nivel de alarma. Esto determina si se genera una alarma al activarse el detector de eventos y la gravedad de la misma. Consulte la página de ayuda general para obtener información sobre las alarmas.

Límite superior

Este detector se aplica sólo a los puntos numéricos. El límite superior es el valor por encima del cual el detector se convierte en activo. El detector se puede configurar opcionalmente con una Duración; el valor del punto debe mantenerse por encima del límite superior durante el tiempo establecido para que el detector se convierta en activo. Los eventos producidos por este detector volverán a la normalidad cuando dejen de cumplirse estas condiciones.

Límite Bajo

Este detector se aplica sólo a los puntos numéricos. El límite inferior es el valor por debajo del cual el detector se convierte en activo. El detector se puede configurar opcionalmente con una Duración; el valor del punto debe mantenerse por debajo del límite inferior durante el tiempo establecido para que el detector se convierta en activo. Los eventos producidos por este detector volverán a la normalidad cuando dejen de cumplirse estas condiciones.

Cambio

   Este detector se aplica a todos los tipos de datos. Se provoca un evento cada vez que cambia el valor del punto. Este tipo de eventos no vuelven automáticamente a la normalidad.

Sin cambios

Este detector se aplica a todos los tipos de datos. Se genera un evento cuando el valor no ha cambiado durante el tiempo establecido por Duración. Esto puede ser útil para detectar cuando el equipo está funcionando mal. Los eventos producidos por este detector vuelven automáticamente a la normalidad cuando ya no se cumple esta condición.

Sin actualización

Este detector se aplica a todos los tipos de datos. Se genera un evento cuando el valor no ha sido actualizado durante el tiempo establecido por Duración. (Un cambio requiere que el valor de la nueva muestra de un punto sea diferente a la anterior. Una actualización sólo requiere que el tiempo de la nueva muestra de un punto sea diferente a la anterior). Esto puede ser útil para detectar cuando el equipo está funcionando mal. Los eventos producidos por este detector vuelven automáticamente a la normalidad cuando ya no se cumple esta condición.

Estado

Este detector se aplica a puntos binarios, multiestados y alfanuméricos. Se activa cada vez que el punto entra en el Estado seleccionado. El detector se puede configurar opcionalmente con una Duración; El valor del punto debe permanecer en el estado durante el tiempo establecido para que el detector se convierta en activo. Los eventos producidos por este detector vuelven automáticamente a la normalidad cuando ya no se cumple esta condición.

Cantidad de cambios de estado

Este detector se aplica a puntos binarios, multiestados y alfanuméricos. Se activa cuando la Cantidad de cambios de estado supera la cantidad establecida dentro del intervalo de tiempo establecido por Duración. Los eventos producidos por este detector vuelven automáticamente a la normalidad cuando ya no se cumple esta condición.

Suma acumulada positiva

Este detector se aplica sólo a los puntos numéricos. El límite positivo es el valor de suma acumulada por encima del cual el detector se convierte en activo, y el Peso es el valor de peso constante a usar para el cálculo de la suma acumulada. El detector se puede configurar opcionalmente con una Duración; la suma acumulada debe mantenerse por encima del límite positivo durante el tiempo establecido para que el detector se convierta en activo. Los eventos producidos por este detector volverán a la normalidad cuando dejen de cumplirse estas condiciones.

Suma acumulada negativa

Este detector se aplica sólo a los puntos numéricos. El límite negativo es el valor de suma acumulada por debajo del cual el detector se convierte en activo, y el Peso es el valor de peso constante a usar para el cálculo de la suma acumulada. El detector se puede configurar opcionalmente con una Duración; la suma acumulada debe mantenerse por debajo del límite negativo durante el tiempo establecido para que el detector se convierta en activo. Los eventos producidos por este detector volverán a la normalidad cuando dejen de cumplirse estas condiciones.
