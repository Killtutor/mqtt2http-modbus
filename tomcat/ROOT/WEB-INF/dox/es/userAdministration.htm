Visión general

Los Usuarios representan las personas que utilizan el sistema. Diversos privilegios se pueden asignar a los usuarios para concederles la capacidad de leer o modificar los datos así como la configuración de parámetros del sistema.

Existen dos niveles básicos de usuarios: administradores (), y usuarios simples (). Los administradores tienen acceso a toda la funcionalidad del sistema. Los usuarios simples tienen varios privilegios dependiendo de la configuración.

Edición de usuarios

Para agregar un nuevo usuario debe hacer clic en el icono . El usuario no será agregado al sistema hasta que sea guardado. Para editar un usuario existente se debe hacer clic en su nombre para mostrar el panel de edición.

Las contraseñas no se pueden recuperar una vez que han sido guardadas.

Un usuario deshabilitado no tiene permisos para entrar al sistema. Para borrar un usuario hacer clic en .

Para verificar la dirección de correo hacer clic en . Esto intentará enviar un correo electrónico a la dirección especificada. Para verificar el número telefónico hacer clic en . Esto intentará enviar un SMS al teléfono móvil especificado.

Los usuarios pueden recibir correos o SMS de alarma sin estar explícitamente agregados a un manejador de eventos seleccionando un nivel diferente de "ninguno" en Enviar alarmas por correo o Enviar alarmas por mensaje de texto, esto provocará que se envíen todas las alarmas con nivel igual o superior al seleccionado. Si la dirección o número ya se encuentran especificados en un manejador de eventos, sólo será enviada una notificación.

Privilegios de usuarios

La casilla de verificación Administrador determina si un usuario tiene privilegios de administrador.

Si un usuario no cuenta con privilegios de administrador, se listarán todas las fuentes de datos que puede compartir para fijar sus privilegios de forma explícita. Si se selecciona la casilla de verificación de la fuente de datos (sólo administradores) se otorgan permisos para compartirla con otros usuarios.

Si la casilla de verificación de una fuente de datos no se activa, se mostrará la lista de todos los puntos que ella contiene. A cada punto se puede fijar individualmente el tipo de acceso (ninguno, leer o modificar).
